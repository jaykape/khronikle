# Khronikle

Khronicle is a learning-driven project that implements key ideas behind *Apache Kafka*, a distributed event streaming platform.

This project isn’t meant to replace Kafka. It’s a hands on exploration of these characteristics Kafka achieve:

- Publish-Subscribe architecture

- Distributed data system with Replication and Partition

- Sequential consistency and offsets

While Kafka is written in Java and Scala, I choose to building Khronikle from scratch in Go.
